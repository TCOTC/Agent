源码与函数导览

下表按文件说明职责；其后按调用链解释关键导出函数。路径均相对于插件仓库的 src/。

文件职责一览

文件 职责
index.ts 插件类 Agent：onload 注册 Dock 与设置；onLayoutReady 加载设置；uninstall 清理存储。
agent/chatDock.ts 挂载 Dock UI、工作集 CRUD 展示、聊天渲染、发送 / 停止、调用 runAgentLoop。

agent/openaiClient.ts openAIChatCompletion：组装请求体、映射消息格式、解析 choices[0].message。
agent/builtinTools.ts 工具定义 getBuiltinToolDefinitions、OpenAI 格式转换 toolsToOpenAIFormat、分发执行 runBuiltinTool。
agent/kernelExecutor.ts createFetchSyncKernelExecutor：fetchSyncPost 适配 KernelExecutor。
agent/storage.ts 存储键常量、默认值、normalizeSettings / normalizeWorkset。
agent/types.ts 共享类型：ToolDefinition、AuditEvent、KernelExecutor、ChatMessage 等。
agent/confirmUtil.ts confirmPromise：思源同步 confirm 的 Promise 封装。
index.scss Dock 内插件样式（BEM 风格类名）。
i18n/*.json Dock 与设置项文案。

runAgentLoop（agent/agentLoop.ts）

作用：实现「LLM ↔ 工具」多轮闭环，直到模型不再请求工具或达到轮次上限。

主要参数（RunAgentLoopParams）：

  • llm：baseUrl、apiKey、model。
  • allowSqlTool：是否把 SQL 工具加入模型可见工具列表。
  • worksetRootIds：Set<string>，工具层用于鉴权。
  • messages：对话历史，函数会原地 push user / assistant / tool（不含每轮临时拼上的 system）。
  • userText：本轮用户输入。
  • signal：AbortSignal，停止按钮触发中止。
  • onAudit：审计回调。
  • kernel：可选，默认 createFetchSyncKernelExecutor()。
  • systemExtra：附加到系统提示的字符串。

副作用：修改 messages；多次网络请求；可能弹出写确认（经 confirmWrite 闭包）。

buildSystemPrompt

把当前工作集 rootId 列表写进系统提示，并声明三条规则：仅访问授权文档、写入需用户确认、回答简洁优先只读工具。

openAIChatCompletion（agent/openaiClient.ts）

将内部 ChatMessage[] 通过 sanitizeForApi 转成供应商期望 JSON；附加 tools（由 getBuiltinToolDefinitions 来）、tool_choice: "auto"。 处理 reasoning_content 以便兼容带思考链的模型回传要求。

getBuiltinToolDefinitions / toolsToOpenAIFormat

前者返回带风险元数据的工具 schema（含 risk、needsWriteConfirm、source）；后者去掉扩展字段，映射为 OpenAI tools[] 项。

runBuiltinTool

根据 name 分支调用内核 API；读操作前 ensureRootInWorkset；写操作前 ctx.confirmWrite； SQL 在函数内单独 confirmPromise。统一返回字符串（多为 JSON 字符串），作为 tool 消息的 content。

mountAgentDock（agent/chatDock.ts）

参数为 (plugin, dockElement)。负责：初始 HTML 模板、加载设置与工作集、渲染消息列表、绑定发送与停止、 在发送路径组装 runAgentLoop 参数。移动端与桌面 Dock 外壳由 index.ts 区分，但挂载点均传入此函数。

normalizeSettings / normalizeWorkset

从持久化读出的弱类型 JSON 规整为稳定结构，避免字段缺失导致运行期异常。

confirmPromise

把思源 UI 的确认 / 取消回调转成 Promise<boolean>，便于在 async 工具里 await。
