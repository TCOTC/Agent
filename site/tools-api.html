内置工具与内核 API

下表对应 builtinTools.ts 中注册的工具名：参数含义、风险、触发的思源 API，以及返回给模型的大致 JSON 形态。

工具总表

工具名 风险 用户确认 思源 API
siyuan_get_block_info 读 无 POST /api/block/getBlockInfo
siyuan_read_doc 读 无 POST /api/filetree/getDoc（mode=0, size=512）
siyuan_search_blocks 读 无 POST /api/search/fullTextSearchBlock
siyuan_append_markdown 写 写入确认 POST /api/block/appendBlock（dataType=markdown）
siyuan_update_markdown 写 写入确认 POST /api/block/updateBlock（dataType=markdown）
siyuan_sql_query SQL 每次 SQL 确认；且需在设置中开启 allowSqlTool POST /api/query/sql

siyuan_get_block_info

  • 参数：id 块或文档 ID。
  • 鉴权：ensureRootInWorkset，块所属 rootID 须在工作集。
  • 返回：{ code, data, msg } 的 JSON 字符串，含 box、path、rootID 等。

siyuan_read_doc

  • 参数：id 必须为文档根块 ID且在工作集中。
  • 截断：HTML 内容长度超过 MAX_DOC_SNIPPET（14_000）时截断并标记 truncated: true。
  • 返回字段示例：rootID、box、path、blockCount、content。

siyuan_search_blocks

  • 参数：query；可选 page（默认 1）、pageSize（默认 16）、method（0 关键字 / 1 查询语法 / 3 正则）。
  • 范围：对工作集中每个 rootId 调 getBlockInfo 拼出 paths，限定搜索路径。
  • 边界：工作集为空时返回错误 JSON，避免全库盲搜。

siyuan_append_markdown

  • 参数：parent_id、markdown。
  • 鉴权：父块所在文档根在工作集。
  • 确认文案：包含 parent 与 markdown 长度，便于用户判断规模。

siyuan_update_markdown

  • 参数：id（目标块）、markdown。
  • 鉴权：目标块所在文档根在工作集。

siyuan_sql_query

  • 前置：allowSqlTool 为真才会出现在模型工具列表；内核侧仍有权限与行数限制（见思源文档）。
  • 确认：弹窗展示 SQL 片段（最长约 800 字符）。

扩展新工具时的检查清单

  1. 在 getBuiltinToolDefinitions 增加 schema 与 risk / needsWriteConfirm。
  2. 在 runBuiltinTool 增加分支：先做工作集或角色校验，再执行 kernel.post。
  3. 若返回结构化数据，保持字符串化 JSON，便于模型在下一轮解析。
  4. 评估是否需要新的设置项或审计事件类型。
