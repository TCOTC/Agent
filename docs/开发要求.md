# Agent 插件开发要求

> 文档版本：v1.0  
> 日期：2026-05-28  
> 来源：插件开发对话中明确的产品目标、技术约束与迭代反馈汇总。

---

## 一、产品定位

- 打造 **思源笔记专用** 的 Agent 工具，体验对标 **Cursor 侧栏**、**Notion Agent** 等顶级产品。
- 证明 **开源方案** 在笔记 Agent 场景下不输付费产品。
- **不分阶段交付**：直接开发到最终形态；停下来时应是可用的完整产品，而非半成品 demo。
- 代码要求 **人类友好、架构清晰、精炼专业**。
- Agent 底座思路参考 [pi](https://github.com/earendil-works/pi)。

---

## 二、范围与约束

| 项 | 要求 |
|---|---|
| 平台 | 仅 **桌面端**（含通过浏览器访问思源的开发/使用场景） |
| 语言 | 仅 **中文**，不做 i18n，文案直接写在代码里 |
| 插件名 | 保持 **Agent** |
| API 提供商 | 暂仅 **DeepSeek**，完全使用其官方接口能力 |
| OpenAI 兼容层 | 不必追求通用 OpenAI API；DeepSeek 相关逻辑可写死 |
| 模型列表 | 通过 DeepSeek API **动态获取** |
| Tool 输出上限 | **14000** 字符 |
| Git | **适当 commit** 便于回滚；**禁止 push 到远程** |

### 保留与重写

- **保留**：`agentLoop` 核心循环、Markdown **流式渲染**逻辑。
- 其余模块可按最终架构 **大规模重写**。

---

## 三、交互与 UI

### 总体目标

- 侧边栏样式与交互 **完全贴近 Cursor**。
- 可参考 [siyuan-agent-demo](https://github.com/TCOTC/siyuan-agent-demo)（作者早期半成品，侧栏仿 Cursor）。
- 可参考社区 [siyuan-plugin-copilot](https://github.com/Achuan-2/siyuan-plugin-copilot)，但不以其为唯一标准。
- 要求 **顶级、愉悦、丰富、强大** 的使用体验，而非功能堆砌的 demo。

### 布局与滚动（已验证的 UX 要求）

- 对话消息区必须 **正常显示** 用户输入与 AI 流式输出。
- 纵向滚动收敛到 **`agent-main__body`**，不要整个侧边栏都成为滚动容器。
- **宽表格** 不得撑宽对话区；表格应 **单独横向滚动**。
- 修复 bug 时采用 **最小有效改动**；无效尝试应及时 **还原**，避免堆叠无效补丁。

### 流式 Markdown

- 流式渲染 **不得重复**（标题、段落、表格等内容不能出现两份）。
- 保留并维护 **封存渲染 + 世代校验** 等竞态防护。

## 四、DeepSeek 集成

- 对接文档：<https://api-docs.deepseek.com/zh-cn/>
- 设置面板配置 API Key（沿用现有存储方式）。
- 支持 DeepSeek 提供的 **思考模式**、**Function Calling**、流式 `reasoning_content` 等能力。
- 模型选择、会话、Token 统计等与 DeepSeek 能力对齐。

## 五、工具链（Tool）

### 通用

- 所有 Tool 输出需 **截断/包装**，遵守 14000 字符上限。
- 插件应能通过 Tool 调用 **思源前端能力**（如 `openTab` 打开文档、跳转到块）。
- 实现 **风险评分** 机制：低风险尽量自动执行，高风险需用户确认。

### 读取

- **`siyuan_read_markdown`**：先获取 Markdown 全文，再按 **行范围**（1-based）截取。
- 导出正文时使用 `addTitle: false`，**不含文档标题行**。
- **`siyuan_read_kramdown`**：按需读取，避免默认全量 Kramdown 浪费 Token。

### 导航

- **`siyuan_open_document`**：打开并 **切换到** 文档/块；`highlight=true` 时滚动高亮，**不将光标定位到块内**。
- **`siyuan_focus_block`**：真正 **聚焦光标到块**，供用户继续编辑。
- 打开行为须对齐思源原生逻辑（如 `checkBlockFold`）：
  - 不对文档根误用 `CB_GET_ALL`，避免面包屑误显示 **「退出聚焦」**。
  - 打开文档应 **可见**，不能只在后台打开。

### 文档编辑

- 大段改写使用 **单一工具 `siyuan_edit_document`**（不要 `propose` + `apply` 两步）。
- Diff 预览弹窗提供 **「拒绝」/「应用」**；用户点 **应用** 后 **立即写入**。
- 单块精确修改使用 **`siyuan_edit_block_kramdown`**，保留块 IAL / ID。
- 写入前须 **验证 Kramdown 有效性**。
- 编辑策略须保护 **块 ID 稳定**，避免删重建导致 **块引用丢失**。

### 标题规范

思源 **单独管理文档标题**，正文 Markdown 不得重复写入标题：

| 场景 | 规则 |
|---|---|
| `siyuan_create_document` | `path` 末段即文档标题；`markdown` 不要再写同名一级标题 |
| `siyuan_edit_document` | 应用前剥离与 `rootTitle` 相同的首行一级标题 |
| `siyuan_read_markdown` | 返回正文，不含标题行 |

### 工具 UI

- 工具调用展示为 **可展开卡片**（参数 + 结果），不能长时间空白。
- 等待 diff 确认等长耗时步骤须显示 **明确提示**。
- 不单独渲染 `role: tool` 消息行（信息合并进 assistant 工具区）。

---

## 六、安全与自动化

- 通过 **置信度 / 风险评分** 控制自动执行范围。
- Agent 应 **尽可能自动运行**，减少低风险操作的人工打断。
- **高风险操作**（如批量删除、整篇替换、SQL 等）必须 **用户确认**。
- 删除等操作需考虑 **引用检测** 与影响范围说明。

---

## 七、可观测性

- **日志** 写入 `data` 目录，便于开发分析与问题排查。
- 实现 **Token 用量统计**。
- 维护 **活动/审计时间线**（工具调用、确认、编辑等事件）。

---

## 八、文档编辑策略（设计原则）

1. **先读后写**，禁止臆测文档内容。
2. 默认用 **Markdown** 读写与 diff；仅在需要保留块结构/引用时用 **Kramdown** 精确改单块。
3. 大改走 **整篇 Markdown diff + 用户确认**；精改走 **单块 Kramdown**。
4. 保持 **块 ID** 与双向链接稳定。
5. 批量或删除操作前说明 **影响范围**。

## 十、已知问题与验收要点（迭代清单）

以下为用户在开发过程中明确反馈、须持续满足的行为：

- [x] 发送消息后用户与 AI 内容均可见
- [x] 流式 Markdown 不重复渲染
- [x] 宽表格不撑宽侧栏
- [x] `agent-main__body` 为滚动容器
- [x] 文档编辑合并为单工具，弹窗「拒绝/应用」
- [x] 创建/编辑文档不写重复标题
- [x] 工具卡片有参数、结果与等待提示
- [x] `open_document` 正常展示文档，不误进块聚焦模式
- [ ] 持续对标 Cursor 侧栏的完整功能与交互细节
- [ ] 功能广度与稳定性达到「世界级 Agent 产品」预期（长期目标）
